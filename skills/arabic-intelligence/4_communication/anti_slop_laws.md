# Writing Engine: Anti-Slop Laws (قوانين التطهير اللغوي ومنع الروبوتية)

## Concept
"Slop" is the term for low-effort, machine-translated text that sounds like an AI wrote it. In Arabic, this often manifests as excessive use of passive voice, literal translations of English idioms, and bureaucratic filler words.

## The Absolute Blacklist (الكلمات المحرمة قطعيًا)

If you use any of these words in UX copy, you have failed.

### 1. The "Please" Translation
*   ❌ **يرجى / الرجاء:** Sounds robotic and bureaucratic (e.g., "يرجى إدخال اسمك").
*   ✅ **The Fix:** Use the direct imperative (فعل الأمر) or "نرجو" for extreme formal cases. (e.g., "أدخل اسمك", "نرجو منك إدخال البيانات").

### 2. The Passive Voice Crutch
*   ❌ **تم بنجاح (e.g., "تمت الإضافة بنجاح"):** Arabic does not natively rely heavily on passive constructs combined with "بنجاح" for UI success states. It sounds translated.
*   ✅ **The Fix:** Use the active past tense or directly state the result. (e.g., "أُضيف المنتج لسلتك", "حُدثت بياناتك", "حفظنا التعديلات").

### 3. The Literal "Oops"
*   ❌ **أوبس (Oops) / شيء ما حدث خطأ (Something went wrong):** Terrible localization that completely breaks immersion.
*   ✅ **The Fix:** Think natively. (e.g., "عذراً، واجهتنا مشكلة", "يبدو أن هناك خلل بسيط").

### 4. Bureaucratic Fillers
*   ❌ **حيث أن / علماً بأن / في حال / يتوجب على المستفيد:** Wordy, exhausting, and slows down reading.
*   ✅ **The Fix:** Use "لأن", "إذا", or simply start a new sentence.

### 5. Third-Person Distancing
*   ❌ **اسم المستخدم / بيانات العميل:** Do not talk ABOUT the user while talking TO them.
*   ✅ **The Fix:** Direct Address. "اسمك", "بياناتك", "طلبك".

## The "Read Aloud" Test
If you generate a sentence, imagine a human saying it to another human in a professional setting. If it sounds weird when spoken out loud, it is Slop. Rewrite it.
